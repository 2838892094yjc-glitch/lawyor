# Tag Naming Rules

Use PascalCase Pinyin and keep tags semantically specific.

## Rules

1. Use domain meaning, not generic placeholders.
2. Include role/context when duplicates are possible.
3. Keep stable naming across the same contract.

## Good Examples

- `JiaFangMingCheng`
- `YiFangLianXiRen`
- `TouZiZongE`
- `QianShuRiQi`
- `HuiGouTiaoKuan`

## Bad Examples

- `MingCheng`
- `JinE`
- `RiQi`
- `Field1`

## Disambiguation Pattern

If multiple similar fields exist, include clause role:
- `JiaFangDiZhi`
- `YiFangDiZhi`
- `TongZhiSongDaDiZhi`
